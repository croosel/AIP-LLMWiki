> **店铺**: LILIS — 婚庆派对用品 · 2026-05-18

## REASON · 多方案效率建模

### 方案矩阵：4 种工作流

| 方案 | 操作路径 | 单品耗时 | 日耗时 (5个) | 新品标记 | ID 变化 | 历史数据 |
|------|---------|---------|------------|---------|---------|---------|
| **A · 全新直上** | 从零创建 → 发布 | 18-22 min | 90-110 min | ✅ 是 | 新 ID | 无 |
| **B · 旧品编辑** | 下架 → 修改 → 重新上架 | 8-12 min | 40-60 min | ✅ 是 | **保留原 ID** | **保留** |
| **C · 裂变克隆** | 复制已有品 → 改标题+属性 → 发布 | 6-10 min | 30-50 min | ✅ 是 | 新 ID | 无（但有模板） |
| **D · 直接编辑（不下架）** | 在线修改 → 保存 | 5-8 min | 25-40 min | ❌ 否 | 原 ID | 保留 |

### 关键发现

```
┌──────────────────────────────────────────────────────────────┐
│                                                                  │
│   ⚠️  方案 B（下架→编辑→重上）是黄金方案                         │
│                                                                  │
│   1. 平台识别为"新品" → 享 90 天新品流量扶持                     │
│   2. 保留原产品 ID → 可选恢复历史点击/转化数据（如有）             │
│   3. 无需从零填属性 → 效率高 2-3 倍                               │
│   4. 可同时修改关键词 → 匹配行业定向征品要求                       │
│                                                                  │
└──────────────────────────────────────────────────────────────┘
```

## REASON · 协同效应量化分析

### 场景设定

- **目标**: 每日上 5 个定向甄品
- **约束**: 要把 126 个普通品逐步替换掉
- **假设**: 5 个定向甄品 = 每日产出定额

### 场景 1：分离操作（现状 / 未协同）

```
时间线:
  08:00-09:40  定向甄品 ×5 全新直上    (方案 A, 100 min)
  09:40-10:00  找到 5 个普通品下架      (方案 A, 20 min)
  ─────────────────────────────────────────────
  日总耗时: 120 min    月耗时: 40 h (按 20 个工作日)
  月度产效: 100 个新品 / 月
  126 普通品清空需: 约 25 个工作日 (5 周)
```

### 场景 2：半协同操作（只用裂变法）

```
时间线:
  08:00-08:40  定向甄品 ×5 裂变克隆      (方案 C, 40 min)
  08:40-08:55  找/下架 5 个普通品         (15 min)
  08:55-09:00  5 个普通品裂变重新发布    (方案 C, 0 min — 已包含)
  ─────────────────────────────────────────────
  日总耗时: 60 min    月耗时: 20 h
  月度产效: 100 个新品 + 100 个普通品替换 / 月
  126 普通品清空需: 约 13 个工作日 (2.6 周)
  💰 效率提升: 50% 时间节省
```

### 场景 3：全协同操作（→ 最优方案）⭐

```
策略: 从 126 个普通品中筛选，定向修改为"定向甄品"

时间线:
  08:00-08:15  数据参谋导出普通品清单，AI 匹配征品要求
               筛出 5 个"普通品 → 可改造为定向甄品"的候选
  08:15-08:55  逐一下架 → 修改(标题+属性+主图+卖点) → 重新发布
               (方案 B, 8-12 min/个 ×5 = 50 min)
  08:55-09:00  确认 5 个新品发布成功, 检查橱窗/广告状态
  ─────────────────────────────────────────────
  日总耗时: 55 min    月耗时: 18.3 h
  月度产效: 100 个普通品→优品改造 + 100 个替换 / 月
  126 普通品清空需: 约 13 个工作日 (2.6 周)
  💰 效率提升: 54% 时间节省
```

### 三种场景流量模型对比

| 场景 | 月产新品 | 额外流量增益 | 时间成本 | 副产物 |
|------|---------|------------|---------|--------|
| 1 · 分离 | 100 | 新品流量 ×100 | 120 min/天 | 无 |
| 2 · 半协同 | 100 | 新品流量 ×100 | 60 min/天 | 126 普通品清空 |
| 3 · 全协同 | 100 | 新品流量 ×100 + **定向甄品流量** | **55 min/天** | 126 普通品清空 + 优品基数提升 |

## REASON · 深层平台规则拆解

### 为什么"下架→编辑→重上"比"全新直上"更好？

| 维度 | 下架→编辑→重上 (B) | 全新直上 (A) |
|------|-------------------|------------|
| **新品标记** | ✅ 重新发布时间重置，触发 90 天新品 | ✅ |
| **产品 ID** | 保留 | 生成新 ID |
| **历史信号** | 若之前有转化，保留关联 | 从零开始 |
| **属性填写** | 已有基础字段，只需改关键差异 | 全部从零填 |
| **关键词覆盖** | 继承 + 新增 | 全新布局 |
| **行业征品匹配** | 在现有品类框架上精准对齐 | 需重新匹配 |
| **重复铺货风险** | **风险**（需充分差异化） | 无 |

### 123 个普通品筛选举例（基于现有数据）

> 根据 `AIP_Alibaba_Integration.md` § Function 2 的诊断逻辑，以下 3 类普通品改造价值最高：

| 普通品类型 | 筛选条件 | 改造为定向甄品策略 | 预计自然流量提升 |
|-----------|---------|------------------|---------------|
| **高曝光零询盘** | 曝光 > 30/周 且 询盘=0 | 改主图为婚礼场景 + 加品牌前缀 + 匹配征品要求 | +40-60% |
| **中等曝光有信号** | 曝光 15-30/周 且有 TM 点击 | 全量匹配行业征品类目 + 价格对齐第一页 | +80-120% |
| **零曝光沉底品** | 曝光=0（≥30天无曝光） | 下架后换关键词重新发布，不保留原主图 | +200-500% |

## REASON · 风险矩阵

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| 重复铺货被判定 | 中 | 产品降权 / 下架 | 改标题 ≥50%、换主图、差异化属性 |
| 定向征品类目变更 | 低 | 已发布品不再匹配 | 每周复查 MA 后台征品更新 |
| 老客丢失 | 低 | 原产品被下架后找不到 | 仅在 30 天零效果品上操作 |
| 编辑匆忙导致属性错误 | 中 | 审核不通过/发布失败 | AI 辅助填充 + 批量检查工具 |

## ACT · 最优协同方案（全协同模型）

### 🏆 每日 SOP

```
┌─────────────────────────────────────────────────────────────┐
│                     每日 55 分钟 · 5 步                      │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  Step 1 · 数据提取 (5 min)                                    │
│    ├─ MA 后台 → 产品参谋 → 导出普通品清单                     │
│    ├─ MA 后台 → 行业定向征品 → 查看今日推荐类目                │
│    └─ 用 AI 工具交叉匹配：哪些普通品类目在征品列表中           │
│                                                               │
│  Step 2 · 5 个候选品筛选 (5 min)                              │
│    ├─ 优先: 已有曝光但零询盘的普通品 (曝光>20/周)              │
│    ├─ 次优: 曝光低但有 TM 信号的普通品                         │
│    └─ 排除: 近 30 天有订单记录的商品（保留，不操作）           │
│                                                               │
│  Step 3 · 逐一下架+修改 (40 min · 8 min/个)                    │
│    ├─ ① 下架原普通品                                          │
│    ├─ ② 修改标题: 品牌 + 核心词 + 属性 + 场景 (128 字符)       │
│    ├─ ③ 匹配定向征品: 根据 MA 推荐调整类目/属性/价格           │
│    ├─ ④ 更换主图: 婚礼场景图 (不是白底图)                     │
│    ├─ ⑤ 填写商品卖点: 充分利用 2000 字符（AI Mode 搜入词库）  │
│    └─ ⑥ 重新发布                                             │
│                                                               │
│  Step 4 · 确认发布 (3 min)                                    │
│    ├─ 检查 5 个新品是否在"新品"列表中                          │
│    ├─ 确认无违规提示                                           │
│    └─ 确认标题/主图/卖点三项无误                               │
│                                                               │
│  Step 5 · 入橱窗候选池 (2 min)                                │
│    └─ 将 5 个新品标记为"待观察入橱窗"（7 天后评估数据）        │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

### 🎯 月目标与里程碑

| 周 | 累计改造普通品 | 优品预期新增 | 自然曝光预期 | 备注 |
|----|-------------|-----------|------------|------|
| W1 | 25 | +3 (部分改造品30天后成优品) | 3,500/周 (+21%) | 仅橱窗+改造 |
| W2 | 50 | +5 | 3,800/周 | |
| W3 | 75 | +8 | 4,200/周 | |
| W4 | 100 | +10 | 4,600/周 (+59%) | 普通品减至 26 |

> 注意：产品等级评估有 15-30 天延迟。改造后的产品需经过平台数据计算才能被评定为优品。上述"优品预期新增"是 30 天后的预期值（今日改造 → 30天后评估）。

### ⚙️ 自动化辅助建议

| 环节 | AI 辅助方式 | 提速 |
|------|-----------|------|
| 普通品筛选 | AI 读数据参谋导出表 → 按"定向征品匹配度"排序 | -15 min |
| 标题生成 | AI 根据原品+征品要求 → 输出 3 个标题方案 | -2 min/个 |
| 卖点生成 | AI 读行业热词 → 输出 2000 字卖点 | -5 min/个 |
| 主图生成 | AI 根据原品白底图 → 生成婚礼场景图 | -3 min/个 |
| 发布确认 | AI 检查 5 个新品 → 列出差异 | -2 min |

> 配置自动发品脚本 / 裂变发品工具可进一步将日耗时压缩至 **25-30 分钟**。

## ACT · 最终结论

### 三重效益叠加

```
                   ┌──────────────┐
                   │  入选征品    │  ← 平台买家需求匹配 → 转化率更高
                   │  (定向甄品)  │
                   └──────┬───────┘
                          │
          ┌───────────────┼───────────────┐
          │               │               │
    ┌─────▼─────┐  ┌──────▼──────┐  ┌─────▼─────┐
    │ 新品流量  │  │ 优品达标签  │  │ 自然排名  │
    │ 90天扶持  │  │ 类目/价格力 │  │ 橱窗加成  │
    └───────────┘  └─────────────┘  └───────────┘
          │               │               │
          └───────────────┼───────────────┘
                          │
                   ┌──────▼──────┐
                   │ 三源流量叠加 │  → 自然流量最大化
                   │ 1 + 1 + 1 > 3│
                   └─────────────┘
```

### 一句话结论

> **用「下架普通品 → 按定向征品要求修改 → 重新发布」的方案，5 个定向甄品的日工可从 120 分钟压缩到 55 分钟，同时获得新品流量 + 定向征品流量 + 优品通路三重叠加。这是最小时间成本产生最大自然流量的最优路径。**
