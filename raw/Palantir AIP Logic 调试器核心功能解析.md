在 Palantir AIP Logic 的调试器（Debugger）面板中，系统通过追踪大语言模型（LLM）的**思维链（Chain-of-thought, CoT）**，使开发者能够透明地观察模型在每一步推理过程中调用的具体工具 1, 2。  
根据来源，AIP Logic 调试器支持追踪以下三类核心支撑工具的使用情况：

* **查询对象 (Query Objects)**：  
* 追踪 LLM 如何在本体（Ontology）中自主搜索所需的特定业务数据 2。  
* 通过此工具，开发者可以核实 AI 是否准确地找到了决策所需的实体及其属性 2。  
* **计算器 (Calculator)**：  
* 追踪 LLM 何时调用专门的计算工具来进行数值运算 2。  
* 使用此工具是为了获取精确值，从而弥补 LLM 自身在处理复杂数学逻辑时可能产生的偏差或幻觉 2。  
* **应用行动 (Apply Action)**：  
* 追踪 LLM 根据推理结果对本体数据执行的修改指令，或向外部系统（如 ERP）发起的写回操作 2。  
* 这是将 AI 推理转化为现实世界行动的关键步骤，调试器允许开发者在操作正式生效前验证其逻辑的正确性 2。

**调试器的核心价值：**

* **可视化思维链**：调试器会详细展示逻辑函数中每个组件块的执行步骤，揭示 LLM 的“思考过程” 1。  
* **排除幻觉**：通过查看这些工具调用的细节，开发者可以确保 AI 的决策是基于本体中的真实事实而非凭空捏造 1, 3。  
* **中间过程监控**：调试器与 AIP Evals 框架协同工作，支持对这些中间参数和步骤进行定量管理和验证 2。
