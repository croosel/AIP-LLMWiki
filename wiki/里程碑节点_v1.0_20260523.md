## 2. 核心技术资产 (Core Tech Assets)

### A. SPE 指令协议 (Shortest Path Execution)
- **定义**：全链路 P0 化。不仅是“写”数据（Act），“搜” ID（Sense）和“验”结果（Eval）也必须通过后台 API 完成，彻底终结“网页转圈”。
- **闭环标准**：物理状态回读（Read-back）是唯一成功判据。

### B. 双 Agent 协同模型 (Conductor-Executor)
- **指挥官 (Accio)**：负责提取 Wiki 灵魂（Ontology 变量），生成 JSON 指令包。
- **执行官 (Alibaba Agent)**：负责利用 Native API 权限执行物理写入。

### C. 变量化驱动 (Zero-Prompt Injection)
- 实现了将品牌手册、SEO 公式、图片资产 URL 转化为执行变量，减少了 95% 的人类手工输入。

## 3. Ontology 本体模型更新 (v1.1)

- **New Object: `MultiAsset_Optimization`**
    - `Properties`: `title_128char`, `image_5_service`, `image_6_us_warehouse`.
    - `Logic`: 强制执行“标题+副图”同步更新，确保全链链路一致性。
- **Constraint: `Length_Hard_Limit`**：标题强制约束在 128 字符内，超出则自动触发语义压缩。
