# AI 前沿开发工程师 (AI-FDE)

## 定义
**AI 前沿开发工程师 (AI Frontier Development Engineer, AI-FDE)** 是 Palantir AIP 提供的低代码应用生成工具。业务人员用自然语言描述需求，AI-FDE 自动生成对应的 React 应用、本体配置或 Python 脚本，极大降低 AI 集成的技术门槛。

## 核心属性
| 属性 | 值 |
|------|-----|
| 英文全称 | AI Frontier Development Engineer |
| 输入 | 自然语言需求描述 |
| 输出 | React 应用 / 本体配置 / Python 脚本 |
| 目标用户 | 业务专家（调度员、临床医生、运营人员） |
| 核心价值 | 绕过中央 IT 瓶颈，业务自服务 |
| 范式 | 低代码 + AI 辅助开发 |

## 业务用途

### 传统开发 vs AI-FDE
| 维度 | 传统开发 | AI-FDE |
|------|---------|--------|
| 发起人 | 业务提需求 → IT 排期 | 业务直接用自然语言描述 |
| 周期 | 数周-数月 | 分钟-小时 |
| 技术门槛 | 需要开发团队 | 业务人员可操作 |
| 产出 | 定制应用 | React 应用 + 本体配置 + 脚本 |
| 迭代 | 变更流程长 | 即时修改即时生效 |

### AI-FDE 三阶段流程
1. **需求表达**：业务人员用自然语言描述「我需要一个展示患者等待时间的仪表板」
2. **自动生成**：AI-FDE 生成 React 前端 + 本体查询配置 + 部署脚本
3. **即时验证**：生成结果可立即在 AIP 环境中预览和测试

### 规模化价值
- 从「想法」到「生产应用」的转化速度提升 10-50 倍
- 业务专家直接定义工具 —— 他们最懂业务痛点
- 中央 IT 从「制造者」转型为「治理者」

## 外贸应用
> 参见 [[Accio-Work运营工具链]] · [[运营数据分析SOP]]

对 LILIS 的映射：
- **概念对应**：Accio-Work 的发品工具、分析工具本质上是 AI-FDE 理念的 Alibaba 特化实现
- **运营应用场景**：
  - 运营说「筛选曝光>30 零询盘的产品」→ AI 自动生成筛选仪表板
  - 运营说「生成 5 个 Modern Romantic 标题方案」→ AI 自动输出 5 个 128 字符标题
- **未来愿景**：LILIS 运营人员无需等待开发排期，自然语言描述即可构建自定义数据看板

## 常见问题
**Q: AI-FDE 生成的应用能直接用于生产吗？**
A: 生成结果需经过 AIP Evals 评估和权限审核，但核心逻辑和 UI 框架可直接投入使用，大幅缩短从原型到产线的距离。
