# AIP 评估套件深度

## 定义
**评估套件 (Evaluation Suites)** 是 AIP Evals 框架中用于对 Logic 函数进行定量、系统化验证的核心工具。它提供测试用例编写、中间参数评估、方差分析、跨模型对比等能力，将 AI 决策质量从定性「感觉」转化为定量管理的生产流程。

## 核心属性
| 属性 | 值 |
|------|-----|
| 所属框架 | AIP Evals |
| 评估对象 | Logic 函数 |
| 核心能力 | 5 项（测试用例 / 中间参数 / 方差分析 / 模型对比 / 可视化） |
| 治理目标 | Responsible AI |
| 输出 | 数据集 (Dataset) + 指标仪表板 (Metrics Dashboard) |

## 业务用途

### 五项核心能力

**1. 详细测试用例**
- 模拟真实业务输入，预设期望输出
- 将 AI 建议的本体编辑与「标准答案」对比
- 发布 Logic 函数后可配置评估套件

**2. 中间参数评估**
- 不仅检查最终结果，还验证特定「块 (Block)」的中间输出
- 确保 AI 的中间推理步骤正确，而非靠「猜对」
- 例如：验证「查询对象」块是否找回正确实体

**3. 方差分析 (Variance Analysis)**
- 在多次运行中执行相同 Logic 函数
- 检查 AI 决策的一致性
- 识别不稳定行为并定位原因

**4. 跨模型性能对比**
- 同一 Logic 函数对比 GPT-4 vs GPT-3.5
- 选择对特定业务场景准确率最高的模型
- 数据驱动选型，非直觉判断

**5. 指标仪表板可视化**
- 评估结果写入 Dataset 长期存储
- 仪表板图形化展示准确率趋势、方差、Token 消耗
- 支持端到端可观测性

### 与本体治理的协作
- Action 内置验证规则（如「库存不能为负」）
- 评估套件结合本体约束，确保 AI 编辑合规
- 两者互补：Evals 验证逻辑正确性 + Action 验证业务合规性

## 外贸应用
> 参见 [[AIP-Evals]] · [[LILIS伦理审查]] · [[AIP-伦理与治理]]

对 LILIS 的映射：
- **P4P 出价评估**：创建测试用例模拟「ROI<1 → 降低出价」逻辑，验证 AI 建议是否合理
- **标题生成评估**：对比不同模型（GPT-4 vs GPT-3.5）生成的 128 字符标题质量
- **方差分析**：确保同一产品在不同时间生成的优化建议一致

## 常见问题
**Q: 中间参数评估与最终结果评估的区别？**
A: 传统评估只看「结论对不对」；中间参数评估检查每一步推理是否正确。例如 AI 说「降价」，中间参数验证它是否真的查对了竞品价格数据。

**Q: 评估套件运行频率？**
A: 每次修改 Logic 函数后立即运行；生产环境中可设置为定期自动运行，监控模型衰减。
