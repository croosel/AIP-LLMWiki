# LILIS 双轨实施计划：数据驱动的正反馈循环

## 战略框架

### 双轨制定义

```
轨道 1: Alibaba 国际站（信任基建轨道）
  ├─ 角色: 信用积累 + B2B 买家验证 + Accio AI 采购入口
  ├─ 预算: 压缩至维持信保活跃度的最低水平
  ├─ 核心 KPI: 星等级四维 ≥ 4.0 / 信保订单 ≥ 2单/周
  └─ 不追求: 高曝光量 / 高 P4P 消耗

轨道 2: 独立站 + 社媒（品牌增长轨道）
  ├─ 角色: AI 搜索可达性 + 品牌资产 + 客户关系 + 数据所有权
  ├─ 预算: 占总预算 70-80%（其中投流 40% + 内容 30% + 技术 30%）
  ├─ 核心 KPI: AI 代理流量占比 / Email List 增长率 / 独立询盘转化
  └─ 不追求: 短期 ROI（前 6 个月以数据积累为目标）
```

### 正反馈循环机制

```
┌──────────────────────────────────────────────────────┐
│                                                      │
│  数据采集 ←──────────────────────────┐               │
│    ↓                                 │               │
│  态势感知（Ontology + AIP）            │               │
│    ↓                                 │               │
│  决策执行（自动化 / 人工确认）          │               │
│    ↓                                 │               │
│  结果记录 → 反馈 → 修正决策参数 ──────┘               │
│                                                      │
│  每完成一轮循环 → 经验层 +1 模式 → 下次决策更准确       │
│  按周复盘 → 按月调整 → 按季迭代战略                     │
└──────────────────────────────────────────────────────┘
```

## 四阶段实施路线图

### Phase 1: 基础建设期 — 2026 Q3 (7-9月)

**目标**: 双轨上线 + 数据管道贯通 + 首轮结构化 Catalog

| 轨道 | 行动 | 产出 | 预算 |
|------|------|------|------|
| **Alibaba** | 压缩 P4P 至 $10/天<br>补充 Listing 结构化属性<br>维持信保 2单/周 | 15 个 Listing 结构化<br>星等级 ≥ 3.5 | $900/月 |
| **独立站** | Shopify 搭建<br>UCP 合规 Catalog<br>Google Merchant Center Feed<br>Pinterest Business + 10 篇首轮内容 | 独立站上线<br>Catalog 结构化<br>Pinterest 账号活跃 | $300-500/月 |
| **数据** | Ontology 四核心 Object Types<br>Python 数据管道（Alibaba→独立站）<br>GA4 + 独立站 Analytics 打通 | 统一数据结构<br>实时数据流 | $0（自建） |
| **决策** | 首个自动化流程：P4P 出价异常告警<br>每周人工数据复盘（2h） | 态势感知 v0.1 | — |

**Phase 1 KPI**：

| KPI | 当前基线 | Q3 目标 |
|-----|---------|--------|
| Alibaba 信保订单 | 不详 | ≥ 8单/月 |

**Phase 1 反馈循环**：

## 外贸应用：LILIS 正反馈循环的独特优势

### 品类特性使 OODA 循环天然加速

婚庆装饰品类有清晰的季节性，这反而让 OODA 循环更快、更准确：

```
Q1（1-3月）→ Q2（4-6月）→ Q3（7-9月）→ Q4（10-12月）
情人节       婚礼旺季       淡季备货       圣诞+新年
              ↑                             ↑
         最高频OODA循环                  第二高频

一年四轮季节变化 = 四轮完整的 OODA 学习周期
比非季节性品类多 2-3 倍的循环次数
→ 经验积累速度 2-3 倍 → 护城河形成更快
```

### 双轨数据的交叉验证

```
Alibaba 数据（广度）       独立站数据（深度）
─────────────────         ─────────────────
P4P 关键词 → 曝光/点击      Google Analytics 完整用户路径
询盘量（总数）               Email 捕获 → 培育 → 转化
信保订单量                  购物车 → 结算
买家国家/行业（粗糙）        买家行为（停留/滚动/点击热点）

交叉验证价值:
  Alibaba 告诉你是谁在买（买家画像）
  独立站告诉你怎么买的（购买路径）
  → 完整的买家旅程 → 更精准的 OODA 推理
```

## 常见问题

**Q: 为什么要同时维持两条轨道，不能完全切到独立站吗？**
A: 不能。在 B2B 场景中，买家在 AI 代理发现你之后，仍然会搜索你的 Alibaba 店铺验证。信保交易记录是独立站无法替代的信任信号。双轨的成本是「多花 $900/月的 Alibaba 维护费」，收益是「独立站的询盘被 Alibaba 信保背书后转化率更高」。

**Q: 如果 Q4 数据显示独立站 ROI 很差，怎么办？**
A: 这正是反馈循环的价值。Q4 的 Google Ads / SEO 数据会告诉你哪个渠道在亏钱——亏的就砍预算，赚的就加。关键是不在 Q3（数据空白期）做决定。Phase 1-2 的核心目标不是 ROI，是积累数据。

**Q: OODA 循环真的需要自动化吗？小团队手动不行吗？**
A: Phase 1-2 手动完全可行（每周 2h 数据复盘）。自动化是在 Phase 3（≥ 5 条高置信规则时）才逐步引入。小团队的优势恰恰是 OODA 循环链路短——从「看到数据」到「做决定」不需要跨部门审批。

**Q: 这个计划的 Budget 估算靠谱吗？**
A: 独立站 Shopify 基础版 $29/月，Google Ads 测试预算 $200/月，Pinterest 免费。Alibaba P4P $10/天已验证可行。数据技术自建（Python + Streamlit 免费）。实际月支出可能低于估算——估算取了安全上界。
