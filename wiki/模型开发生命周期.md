# 模型开发生命周期

## 定义
Palantir Foundry 的**模型 (Model)** 是封装了任意机器学习逻辑的工件。模型可以在数据管道、Ontology 和应用层中复用。Palantir 采取开放策略，模型既可在平台内开发，也可从外部导入（库 / 工件 / 容器 / API），支持从开发→评估→部署→治理→运维的完整生命周期。

## 核心属性
| 属性 | 值 |
|------|------|
| Artifact | Model（封装 ML 逻辑的标准化工件） |
| 生命周期 | 开发 → 集成 → 评估 → 管理 → 测试 → 部署 → 治理 |
| 开发方式 | 平台内开发 / 外部导入 / 外部 API 代理 |
| 治理核心 | Modeling Objectives |
| 评估 | MetricSets + Evaluation Dashboard |
| 部署 | Live Deployment（低延迟） / Batch（定时） |
| 运维 | ML 反馈闭环 + Model Monitoring |

## 开发路径

### 四大路径对比
| 路径 | 适用场景 | 代码量 | 产出 |
|------|---------|--------|------|
| **Model Studio** | 分类/回归/时序预测 + 新手友好 | 零代码 | 训练好的 Model + 实验记录 |
| **Jupyter Code Workspace** | 探索性分析 + 快速迭代 | Python 脚本 | Model Adapter + Model Artifact |
| **Code Repository** | 生产级训练管道 | 完整代码工程 | 可重现的 Pipeline |
| **外部导入** | 已有模型/第三方 ML 平台 | 封装代码 | Model Adapter 代理 |

## 开发路径详解

### 路径 1：Model Studio（零代码）
```
选择任务类型 (分类/回归/时序预测)
  → 选择输入 Dataset
  → 配置高级选项（Class Weights, Outlier Removal, Feature Override）
  → 训练 → 自动实验追踪 + Metrics
  → 部署
```
- 支持框架：内置生产级训练器
- 特色：Smart Defaults + Guided Workflows
- 数据溯源：全链路 Lineage

### 路径 2：Jupyter Code Workspace
```
Create Jupyter Workspace
  → Import Data (Foundry Dataset → pandas)
  → Write ML Code (scikit-learn / PyTorch / XGBoost / LightGBM)
  → Implement Model Adapter (input schema + output schema + predict method)
  → Publish to Foundry
```
- 仅支持单节点 ML 框架（不支持 SparkML）
- 输出：`palantir_models` 包格式的 Model Artifact

### 路径 3：Code Repository
- 代码工程化，支持 Git 版本控制、CI/CD
- 适合团队协作 + 可重现的 Pipeline
- 可在 Pipeline Builder 中调度执行

### 路径 4：外部导入
| 方式 | 说明 |
|------|------|
| 导入模型权重 | 从文件系统导入 `.pkl` / `.pt` 等 |
| 容器模型 | 打包为 Docker 容器 → `ContainerModelAdapter` |
| 外部 API 代理 | 代理 Azure ML / SageMaker / Vertex AI / OpenAI |
| 函数接口 | LLM 直接注册为函数（用于 AIP Logic / Pipeline Builder） |

## Modeling Objectives（模型治理中枢）
> 详见 [[模型评估与治理]]

| 功能 | 说明 |
|------|------|
| 模型提交 | 统一入口（含 metadata：训练者、参数、描述） |
| 自动评估 | 每次提交在评估数据集上生成 MetricSets |
| 版本对比 | Evaluation Dashboard 可视化对比多版本 |
| 审批发布 | 治理层：模型需审核后进入生产 |
| CI/CD | 零停机持续部署 |

## 常见问题
**Q: 什么时候用 Model Studio 什么时候用 Jupyter？**
A: 分类/回归/时序预测的快速原型 → Model Studio。需要自定义损失函数、自定义模型架构、复杂特征工程 → Jupyter。

**Q: SparkML 能在 Foundry 里用吗？**
A: Model Asset 不支持 SparkML。如需分布式训练，用 Code Repository + Python Transforms。
