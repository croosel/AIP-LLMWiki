# 模型评估与治理

## 定义
Palantir Foundry 的**模型评估和治理体系**，以 **Modeling Objectives** 为中枢，覆盖模型的自动化评估、版本对比、审批发布、生产运维和 ML 反馈闭环。

## 核心架构

### Modeling Objectives 五层功能
```
1. 治理与权限层 → 模型提交审批、元数据管理
2. 自动化评估层 → 每次提交自动在评估数据集上生成 MetricSets
3. 对比管理层 → Evaluation Dashboard 多版本可视化对比
4. CI/CD 层 → 零停机持续部署
5. 反馈闭环层 → 生产数据回流 → 模型监控 → 自适应
```

## MetricSets（评估指标集）

### 定义
`MetricSet` 封装了单次模型评估的全部数值指标、图像和图表。包含：
- 模型及其版本引用
- 评估数据集及其事务版本
- 所有评估库的子集指标（Overall + Subsets）

### 特性
| 特性 | 说明 |
|------|------|
| 数据不可变 | 每次评估绑定评估数据集的特定事务版本 |
| 子集评估 | 支持按维度（如地区、品类）细分指标 |
| 可视化对比 | Dashboard 中对比多版本、多模型的 MetricSets |
| 自动生成 | Modeling Objective 提交自动触发评估 Pipeline |

## 评估配置

### 评估库 (Evaluation Library)
| 库 | 用途 |
|----|------|
| 回归指标 | RMSE / MAE / R² |
| 分类指标 | Accuracy / Precision / Recall / F1 / ROC-AUC |
| 自定义库 | 用户定义的指标计算逻辑 |

### 评估子集 (Evaluation Subsets)
```
Overall（全部数据）
  ├── 按品类: Wedding / Party / Christmas
  ├── 按地区: US / EU / AU
  └── 按价格区间: <$10 / $10-$50 / >$50
```

子集用于检测模型的**公平性、可解释性**，特别是对受保护类别。

### 子集配置方式
| 字段类型 | 分桶策略 |
|---------|---------|
| String | 每个唯一值一个子集 |
| Numeric | Fixed Width（固定宽度）或 Range Cut-offs（自定义区间） |
| Multi-field | 多字段组合子集（笛卡尔积） |

## 评估与治理流程

### 完整 Pipeline
```
模型开发完成
  → 提交到 Modeling Objective
  → 自动评估 Pipeline 启动
    → 在评估数据集上运行推理
    → 生成 MetricSets (Overall + Subsets)
    → Evaluation Dashboard 更新
  → 人工审核 (对比历史版本)
  → 审批通过 → 部署到生产
  → 持续监控 → ML 反馈闭环
```

### 元数据管理
```yaml
Model Submission Metadata:
  owner: 提交者
  description: 模型描述
  training_params: {learning_rate, epochs, ...}
  custom_fields: {experiment_id, data_version, ...}
```

## 生产运维

### ML 反馈闭环
```
生产推理 → 生产数据 → 评估 Pipeline → 新 MetricSets
  → 对比历史指标 → 检测性能衰减
  → 触发预警 → 人工/自动重新训练
```

### 模型退化检测
- 持续对比生产数据上的 MetricSets
- 数据漂移检测（输入分布变化）
- 概念漂移检测（输入→输出关系变化）

## 常见问题
**Q: MetricSet 和 AIP Evals 的 Evaluation Suite 有什么关系？**
A: MetricSet 是面向传统 ML 模型的评估（回归/分类指标）。Evaluation Suite 是面向 AIP Logic 函数（LLM 输出）的评估。两者互补，覆盖不同模型类型。

**Q: 评估子集太多会怎样？**
A: 子集数量 = 字段 × 桶数，组合字段呈指数增长。大量子集会显著增加评估 Pipeline 构建时间，建议控制 <100 个子集。
