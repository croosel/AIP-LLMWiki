# 新品工作流效率模型

## 定义
基于 Palantir AIP **Sense → Reason → Act** 框架，对 LILIS 每日上品工作流进行效率建模与方案择优的方法论。核心目标：**以最小时间成本获得最大自然流量**。

## 核心属性
| 属性 | 值 |
|------|-----|
| 框架 | Sense → Reason → Act |
| 店铺 | LILIS — 婚庆派对用品 |
| 核心矛盾 | 126 普通品（35.5%）无自然流量，占运营空间 |
| 最优方案 | 方案 B（下架→编辑→重上） |
| 日耗时 | 55 min（从 120 min 压缩） |
| 效率提升 | 54% |

## 业务用途

### 四种方案对比
| 方案 | 操作路径 | 单品耗时 | 新品标记 | ID 变化 |
|------|---------|---------|---------|---------|
| A · 全新直上 | 从零创建 → 发布 | 18-22 min | ✅ | 新 ID |
| B · 旧品编辑 | 下架 → 修改 → 重新上架 | 8-12 min | ✅ | 保留原 ID |
| C · 裂变克隆 | 复制已有品 → 改标题+属性 → 发布 | 6-10 min | ✅ | 新 ID |
| D · 直接编辑 | 在线修改 → 保存 | 5-8 min | ❌ | 原 ID |

### 三种场景效率对比
| 场景 | 日耗时 | 月产新品 | 额外增益 |
|------|--------|---------|---------|
| 1 · 分离操作 | 120 min | 100 | 无 |
| 2 · 半协同 | 60 min | 100 | 126 普通品清空 |
| 3 · 全协同（最优）| **55 min** | 100 | 普通品清空 + 优品基数提升 |

## 外贸应用
> 参见 [[P4P投流策略]] · [[商品等级体系]] · [[Alibaba排序逻辑]]

### 每日 55 分钟 SOP
1. **Step 1 · 数据提取 (5 min)**: MA 后台导出普通品清单 + 行业定向征品推荐
2. **Step 2 · 候选筛选 (5 min)**: 优先曝光>20/周且零询盘，排除近 30 天有订单记录
3. **Step 3 · 下架+修改 (40 min)**: 标题(128字符) → 定向征品匹配 → 婚礼场景主图 → 2000字符卖点
4. **Step 4 · 确认发布 (3 min)**: 检查新品列表 + 无违规提示
5. **Step 5 · 入橱窗候选池 (2 min)**: 标记"待观察入橱窗"，7 天后评估

### 风险矩阵
| 风险 | 缓解措施 |
|------|---------|
| 重复铺货判定 → 降权 | 改标题 ≥50%、换主图、差异化属性 |
| 征品类目变更 | 每周复查 MA 后台征品更新 |
| 老客丢失 | 仅在 30 天零效果品上操作 |
| 属性错误/发布失败 | AI 辅助填充 + 批量检查 |

## 常见问题
**Q: 为什么「下架→编辑→重上」比「全新直上」更好？**
A: 前者保留产品 ID 和关联历史信号，已有基础字段仅需改关键差异，效率高 2-3 倍；同时触发 90 天新品标记。

**Q: 新品等级评估多久生效？**
A: 产品等级评估有 15-30 天延迟，改造后需等待平台数据计算。
