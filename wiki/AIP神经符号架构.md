# AIP 神经符号架构

## 定义
Palantir AIP 采用的**神经符号（Neuro-Symbolic）架构**，将大语言模型（LLM）的神经网络推理能力与企业**本体（Ontology）** 的符号化结构逻辑深度集成，使 AI 从「聊天机器人」升级为驱动实际业务操作的「企业 AI 操作系统」。

## 核心属性
| 属性 | 值 |
|------|-----|
| 架构范式 | Neuro-Symbolic（神经符号） |
| 数据基础 | Ontology 本体系统 |
| 核心机制 | 本体增强生成 (OAG) |
| 执行环境 | AIP Logic 低代码开发 |
| 闭环实现 | 动力层 (Kinetic Layer) |
| 质量保障 | AIP Evals 评估框架 |
| 应用构建 | AI-FDE 前沿开发工程 |

## 业务用途

### 六大核心机制

**1. 本体系作为「业务语言」**
- **语义锚定**：本体为 AI 提供名词（患者、订单）和动词（出院、取消）的统一语义
- **封闭世界假设 (CWA)**：本体是企业现实的唯一权威表示，AI 推理边界确定且受控

**2. 本体增强生成 (OAG) 减少幻觉**
- 与 RAG 不同，OAG 将**结构化对象本身**（非文本片段）注入 LLM 上下文
- 基于预定义属性和关系进行确定性推理，大幅降低幻觉风险

**3. AIP Logic：AI 作为执行工具**
- 三类核心工具：查询对象 (Query Objects) / 计算器 (Calculator) / 应用行动 (Apply Action)
- 链式块（Blocks）构建复杂自动化工作流

**4. 动力层 (Kinetic Layer) 闭环操作**
- 写回 (Write-back) 机制通过 Webhooks 同步到 SAP/ERP
- 预提交 (Pre-commit) 验证确保事务一致性

**5. AI-FDE 快速应用构建**
- 自然语言描述需求 → AI 自动生成 React 应用、本体配置或 Python 脚本
- 业务专家直接参与，绕过中央 IT 瓶颈

**6. 严谨的治理与评估**
- 思维链 (CoT) 追踪每个推理步骤
- 评估套件自动化测试、跨模型对比

## 常见问题
**Q: 神经符号架构与纯 LLM 的区别？**
A: 纯 LLM 基于概率生成，可能产生幻觉。神经符号架构将 LLM 的推理能力约束在本体的结构化事实中，每次决策都基于真实的业务对象，输出是确定性的。

**Q: 封闭世界假设 (CWA) 是什么？**
A: 封闭世界假设认为：本体中没有声明的事实即为 false。这与互联网搜索的开放世界形成对比，确保企业 AI 不会基于不存在的数据做决策。
