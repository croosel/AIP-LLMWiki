# 运营数据分析 SOP

## 定义

Alibaba 国际站运营的标准化数据分析流程，覆盖**每日三看**（曝光/点击/询盘）和**每周一清**（下架/重发/复查），辅以二级目标体系指导长期运营节奏。

## 核心属性

### 每日三看

| 表现 | 诊断 | 行动 |
|------|------|------|
| 曝光高但点击低 | 主图不吸引 / [[标题优化黄金公式|标题]]不够精准 | 改主图 + 优化标题 |
| 点击高但询盘少 | 详情页痛点解决方案不足 | 优化前 3 屏 + 添加客户案例 |
| 零曝光产品 | 关键词不匹配 / 价格过高 | 换关键词或降价冲[[信保]] |

### 每周一清

1. 下架 30 天无曝光链接
2. 重新编辑后发布（系统识别为新品给流量）
3. 复查 [[P4P投流完整方法论|P4P 关键词]]报表
4. 按[[橱窗策略|橱窗 SOP]]替换低效橱窗商品

### 二级目标体系

| 目标 | 对标 | 周期 |
|------|------|------|
| 短期 | 长尾关键词下第一页销量 | 1-3 月 |
| 长期 | 核心关键词下第一页销量 | 6-12 月 |

### 建议 Prompt 模板（供 Accio Work）

```
"这是我的阿里巴巴国际站店铺上周的整体数据，请为我分析其中
高曝光低点击和高点击低询盘和TM转化的商品，罗列前20名，
并给我具体的优化建议，以Excel表格的形式输出给我，要求高级排版"
```

## 常见问题

**Q: 30 天无曝光的产品需要删除吗？**
A: 不要删除，下架后重新编辑再发布，系统将识别为新品给予流量扶持。删除会丢失历史数据。

**Q: 数据分析用什么工具？**
A: 数据参谋→产品参谋导出周/月报表，上传 Accio Work 自动分析。也可手动制作 Excel 表格。

## 历史踩坑记录

### [PITFALL] 图片替换 API 直连不可达 (2026-05-24)
- **现象**: 尝试通过 Alibaba API 直连批量替换商品图片，遭遇服务端二次校验失败或权限拒绝。
- **原因**: 常规账户无 OAuth 提权权限，且 Alibaba 服务端对图片上传接口存在严格的二次校验。
- **结论**: 禁止尝试通过 API 直连进行图片替换。必须回退至浏览器自动化路径。
- **参考**: [[API_Direct_Access_Investigation_20260524]]

### [PITFALL] Cookie 获取与 HttpOnly 属性 (2026-05-25)
- **现象**: 在浏览器自动化中，尝试通过 `document.cookie` 获取凭证失败。
- **原因**: 关键凭证（如 `_hvn_login` 等）被标记为 `HttpOnly`，前端 JS 无法直接读取。
- **结论**: 获取 Cookie 时必须处理 HttpOnly 属性，建议通过 HAR 文件导出或录制方式提取，而非简单的 JS 注入。
