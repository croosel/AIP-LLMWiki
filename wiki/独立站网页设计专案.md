# LILIS 独立站网页设计专案

> **专案目标**: 为 LILIS 婚庆派对用品品牌设计并搭建独立站（Shopify），作为[[LILIS双轨实施计划-数据驱动正反馈循环|双轨制]]中"轨道 2: 品牌增长轨道"的核心载体。

---

## 一、战略定位

### 为什么需要独立站？

| 维度 | Alibaba 国际站 | 独立站（Shopify） |
|------|--------------|-----------------|
| 数据所有权 | 平台控制，导出受限 | 100% 自有，GA4 + 邮件全链路 |
| 品牌塑造 | 模板化店铺页，差异化弱 | 完全自主设计，品牌叙事完整 |
| AI 搜索可达性 | 依赖 Accio 算法 | SEO + Schema.org + AI Agent 友好 |
| 客户关系 | 无法直接触达买家 | Email List + Retargeting 闭环 |
| 佣金成本 | 2-5% 交易费 + 信保 | Shopify $29/月 + 2% 支付费 |
| 信任基础 | 平台背书 + 信保 | 需自建（隐私政策/退款保障/评价） |

### 独立站在双轨中的角色

```
轨道 1: Alibaba（信任基建）
  → 信保订单 + 星等级 + 买家验证
  → 独立站引用信保记录增强信任

轨道 2: 独立站（品牌增长）← 本专案
  → AI 搜索可达性 + 品牌资产 + 数据所有权
  → 跨渠道归因 → 验证哪条渠道 ROI 最高
```

**关键联动**: 独立站产品页可链接 Alibaba 店铺，展示信保交易记录，弥补独立站信任短板。

## 二、目标受众与场景

### 主要受众

| 优先级 | 受众 | 场景 | 页面需求 |
|--------|------|------|---------|
| P0 | US/CA Amazon/eBay 卖家 | 批发采购，比价，看 MOQ | 批发价格表、MOQ 说明、快速询盘 |
| P1 | Event/Wedding Planners | 主题套装一站式采购 | 套餐展示、灵感画廊、案例 |
| P2 | 小型零售商 | 稳定补货 | 产品目录、分层定价、库存状态 |
| P3 | 散客/DIY 新娘 | 单次小额购买 | 灵感内容、教程博客、单件购买 |

### 核心用户旅程

```
AI 搜索(Accio/ChatGPT/Perplexity) → 独立站首页/产品页
Google Shopping 广告               → 产品详情页
Pinterest/Instagram 灵感内容       → 博客/灵感画廊 → 产品页
直接访问(品牌词搜索)               → 首页 → 品牌故事
                                    ↓
                              浏览产品 → 加购/询盘 → 邮件培育 → 转化
```

## 三、技术选型

| 维度 | 选择 | 理由 |
|------|------|------|
| **平台** | Shopify (Basic $29/月) | 与[[LILIS跨平台预测分析|跨平台计划]]一致，生态成熟 |
| **域名** | LILISdecor.com | 已在跨平台分析中规划 |
| **主题** | Dawn (免费) 或 Crave | Shopify OS 2.0，JSON 模板，可深度定制 |
| **支付** | Shopify Payments + PayPal | 覆盖北美/欧洲主流支付 |
| **邮件** | Klaviyo (免费 ≤250 联系人) | B2B 邮件自动化最强 |
| **分析** | GA4 + Google Search Console + Microsoft Clarity | 免费全链路，AI referrer 追踪 |
| **SEO** | Shopify 原生 + Schema.org JSON-LD | AI Agent 可解析结构化数据 |
| **社媒** | Pinterest Tag + Instagram Shopping | 婚庆品类 Pinterest ROI 最高 |

## 五、页面设计规范

> 详细设计系统见 [[独立站设计系统]]

### 首页设计要求

| 区域 | 内容 | 设计要点 |
|------|------|---------|
| Hero | 全幅婚庆场景图 + 品牌口号 | 主图突出"一站式"场景（花+拱门+灯光），口号清晰传达品牌价值 |
| 品类导航 | 4 宫格图标 + 品类名 | 图标风格统一，hover 展示子品类 |
| 热门套餐 | 3-4 个 ProductBundle 卡片 | 突出节省比例 + 一站式便利 |
| 信任区 | 信保/认证/评价/美国仓 | 用图标+数据，不用大段文字 |
| Email 订阅 | 1 行输入框 + CTA | 给出订阅理由（获取独家折扣/新品通知） |

### 产品详情页 (PDP) 设计要求

| 区域 | 内容 | 设计要点 |
|------|------|---------|
| 图片区 | 主图 + 5-7 张图 | 场景图优先（婚礼现场），白底图其次 |
| 标题 | 遵循[[标题优化黄金公式]] | 128 字符内，含品牌/件数/场景/差异标签 |
| 价格 | 批发价分层 (Tier 1/2/3) | 清晰展示 MOQ 对应价格 |
| 规格表 | 材质/尺寸/颜色/包装 | 表格化，对 B2B 买家友好 |
| 搭配推荐 | Cross-Sell Chain | 花→拱门→灯光 套餐引导 |
| CTA | "Get Quote" + "Add to Cart" | B2B 询盘 + B2C 购物双按钮 |
| Schema.org | Product + Offer + Review | AI Agent 可直接解析 |

### 博客/灵感页设计要求

| 区域 | 内容 | 设计要点 |
|------|------|---------|
| 文章列表 | 卡片网格，2-3 列 | 封面图 + 标题 + 摘要 + 阅读时间 |
| 文章页 | 图文并茂的长文 | 内嵌相关产品链接，引导购买 |
| CTA | 相关产品推荐 + Email 订阅 | 内容→产品→转化的自然路径 |

## 六、SEO & AI 搜索优化策略

### 传统 SEO

| 策略 | 实施 | 目标 |
|------|------|------|
| 关键词矩阵 | 婚庆品类词 + 长尾场景词 | 参考[[SEO重构工作流v1]]方法论 |
| 页面标题/描述 | 每页独立，含核心关键词 | 60/155 字符限制 |
| URL 结构 | `/collections/` + `/products/` | 简洁、可读、含关键词 |
| 内链策略 | 博客→产品、套餐→单品、相关推荐 | 增加页面权重传递 |
| 图片 Alt | 描述性 Alt + 关键词 | 可访问性 + 图片搜索 |
| Blog 内容 | 每月 2 篇婚庆策划指南 | 长尾流量入口 |

### AI 搜索优化 (AIO)

| 策略 | 实施 | 目标 |
|------|------|------|
| Schema.org JSON-LD | Product/Organization/FAQ/Breadcrumb | AI Agent 结构化解析 |
| UCP 合规 | Product Catalog 符合 Universal Commerce Protocol | 被更多 AI 代理发现 → [[AI搜索对传统电商的冲击]] |
| Google Merchant Center | 产品 Feed 自动同步 | Google Shopping + AI Overview |
| AI Referrer 追踪 | GA4 自定义维度追踪 AI 来源 | 量化 AI 搜索流量占比 |
| 内容可引用性 | 博客用清晰的 H2/H3 + 列表 + 数据 | AI 引用时优先选结构化内容 |

## 七、转化率优化体系

### 漏斗设计

```
曝光(Google Ads/SEO/AI搜索/Pinterest)
  → 点击(CTR 目标: 3-5%)
    → 浏览(页面停留 >30s)
      → 互动(加购/询盘/Email订阅)
        → 转化(下单/成单)
```

### 关键转化抓手

| 抓手 | 触发 | 工具 |
|------|------|------|
| 首单折扣弹窗 | 新用户首次访问 | Klaviyo 弹窗 |
| 询盘即时回复 | 提交询盘后 ≤3h | 邮件自动回复 + 人工跟进 |
| 废弃购物车挽回 | 加购未结账 1h/24h | Klaviyo 自动化流 |
| 浏览挽回 | 看过产品未行动 48h | Retargeting 广告 |
| 批量采购激励 | 加入多件/大额订单 | 阶梯折扣自动应用 |
| 信任强化 | 产品页停留 >15s | 评价/认证/信保弹窗 |

## 常见问题

**Q: 为什么选 Shopify 而不是 WordPress/WooCommerce？**
A: Shopify 维护成本为零（无需管服务器/安全更新），App 生态成熟（Klaviyo/Pinterest/Google 集成一键安装），B2B 批发功能可通过插件实现。LILIS 团队小，需要的是"开箱即用"而非"灵活但需维护"。

**Q: 独立站和 Alibaba 店铺的产品怎么区分？**
A: 独立站侧重品牌展示和套餐组合（[[ProductBundle]]），Alibaba 侧重单品和批发交易。同一产品两端都有，但独立站版本侧重场景化展示，Alibaba 版本侧重规格化展示。

**Q: Phase 1 只上传 15 SKU 够吗？**
A: 15 SKU 足够验证网站功能和初期流量。覆盖 I 层核心品类（人造花 + 拱门 + 气球），每个品类 5 个代表性产品。Phase 3 扩展至 50+。

**Q: 怎么衡量 AI 搜索的效果？**
A: 在 GA4 中设置自定义维度追踪 `referrer = chat.openai.com / gemini.google.com / perplexity.ai / accio.alibaba.com`。这是[[LILIS双轨实施计划-数据驱动正反馈循环|双轨计划]]Phase 1 的核心验证指标。
