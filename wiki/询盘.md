# 询盘 (Inquiry)

## 定义

**询盘 (Inquiry)** 是 Alibaba 国际站买家向卖家发起的采购咨询。询盘来源包括 RFQ（采购需求）、TM（TradeManager 即时通讯）、商品详情页和推荐位。在 [[Ontology]] 模型中，询盘是连接 **Customer** 与 **Product** 的关键桥梁。

## 核心属性

| 属性 | 说明 |
|------|------|
| 来源 | RFQ / TM / 商品详情页 / 推荐 |
| 买家类型 | wedding_planner / party_store / hotel / florist / rental_company / individual |
| AI 分类 | bulk_order / sample_request / custom_design / price_inquiry / spam |
| 紧急度评分 | 0-1，有活动日期 + 明确需求 > 仅问价 |
| 活动信息（婚庆独有） | 活动类型、日期、嘉宾规模 |

## 业务用途

- 判断买家意向质量
- 转化为信保订单
- 收集市场需求信号

## 外贸应用

> 参见 [[AIP-Logic]]、[[Sense-Reason-Act架构]]

AI 分类优先级: `bulk_order > custom_design > sample_request > price_inquiry > spam`

优先回复规则：高紧急度（urgency_score > 0.8）需 24h 内人工跟进。

## 常见问题

**Q: 询盘率和点击率有什么区别？**
A: 点击率 = 点击/曝光，衡量吸引力；询盘率 = 询盘/点击，衡量详情页转化力。

**Q: 什么是 RFQ？**
A: Request for Quotation，买家公开发布的采购需求，卖家可以主动报价竞标。
