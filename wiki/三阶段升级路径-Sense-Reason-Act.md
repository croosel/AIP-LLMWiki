# 三阶段升级路径：Sense → Reason → Act

> **核心理念**: [[Sense-Reason-Act架构]] 不只是一个架构——它是产品从诊断顾问 → 半托管 → 全托管的三阶段演化地图
>
> **演化关键**: 不是按时间升级，是按**数据和信任**升级

---

## 一、Phase 1：Sense + Reason，Manual Act（当前）

### 产品形态：诊断顾问

```
SENSE:  阿里 MCP Connector → 定时拉取数据 → 写入 LLM Wiki 数据层
REASON: Agent 读取 Wiki 规则层 + 数据层 → Ontology 关联推理 → 输出诊断
ACT:    用户看到诊断 → 手动执行（去阿里后台改出价/下架产品）
```

### 交付什么

- 8 个 [[Skills技能集与60-40覆盖|运营 Skills]]
- [[LLM-Wiki统一知识底座设计|LLM Wiki]] 规则层（87 页）
- Wiki 数据层（自动同步店铺数据）
- 每日推送 / 诊断卡片

### 定价

基础版 $29/月，高级版 $99/月。

### 付费理由

「比你雇一个运营便宜，比你手动检查快 10 倍。」

---

## 二、Phase 2：Sense + Reason + Staged Act（半托管）

### 产品形态：半托管运营

```
SENSE:  同上
REASON: [[AIP-Logic]] 函数（自动化 Cron 触发）→ [[AIP-Evals]] 验证通过 → 生成操作建议
ACT:    Staged 模式：
          ├─ 低风险操作（标记低质品、暂停零转化词）→ Auto-apply
          └─ 中高风险操作（调出价、改标题）→ 推送确认 → 用户一键批准

治理:  写入 Audit-Log → 完整可追溯
```

### 关键设计

- 借鉴 [[AI-FDE完整架构]] 的 Staged Mode 思想
- 低风险操作自动执行，高风险保留人工确认
- 所有操作可回滚

### 定价

$99-299/月。

### 付费理由

「你只需要 30 秒确认，剩下的我全做。月费 $299。代运营月费 $1,000。」

---

## 三、Phase 3：Sense + Reason + Auto-apply Act（全托管）

### 产品形态：AI 全托管运营

```
SENSE:  多数据源交叉验证（阿里 + Google Trends + 竞品）
REASON: Logic 函数链（多级推理）+ Evals 高置信度模型 + 跨店铺 Ontology 策略库
ACT:    Auto-apply 为主，仅异常/高金额/策略变更需人工
          ├── P4P 日常调价 → 全自动
          ├── 低质品下架 → 全自动
          ├── 标题微调 → 全自动
          ├── 预算变更 → 人工确认
          └── 选品/定价策略 → 人工决策

治理层:
  - 每个自动操作附带"回滚方案"
  - 异常检测：连续 3 天 ROI 下降 → 自动暂停 → 人工介入
  - 月度治理报告
```

### 定价

$299-999/月 或按效果分成。

### 付费理由

「一个月费顶一个初级运营的年薪。且不会离职。」

## 四、升级触发条件（不是时间驱动）

| 从 | 到 | 必要条件 |
|----|----|---------|
| Phase 1 → 2 | Phase 2 | [[AIP-Evals]] 准确率 > 85% + 用户主动请求执行 ≥ 5 次/月 |
| Phase 2 → 3 | Phase 3 | Staged 操作批准率 > 90% + 零回滚事件 ≥ 60 天 |

**关键原则**：不是时间到了就升级——是数据条件满足了才升级。过早升级会摧毁信任。

---

## 五、经验累积在升级中的作用

| 阶段 | 累积什么 | 对下一阶段的贡献 |
|------|---------|----------------|
| Phase 1 | 每次诊断的结果 + 用户反馈 | 建立 Evals 基线，识别低准确率规则 |
| Phase 2 | 每次 Staged 操作的批准/拒绝模式 | 训练 Auto-apply 的判断边界 |
| Phase 3 | 每次自动操作的 ROI 变化 | 优化全托管策略和风险阈值 |

详见 [[经验累积与竞争壁垒]]。

## 七、为什么现在不直接做 Phase 3

1. **信任未建立**：卖家不会让一个没见过效果的 AI 全权运营店铺
2. **Evals 数据不足**：没有 10,000+ 次诊断验证，全托管风险不可控
3. **离钱快**：Phase 1 只需 4 周 MVP，可以立刻拿到第一笔收入和第一批用户
4. **经验是壁垒**：Phase 1 积累的诊断经验是 Phase 2/3 不可跳过的必要输入
5. **ACCIO Work 平台能力**：目前平台无原生 AIP Logic/自动化执行，需要手动模拟
