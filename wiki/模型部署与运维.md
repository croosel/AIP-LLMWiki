# 模型部署与运维

## 定义
Palantir Foundry 的模型部署和运维体系，包括 **Live Deployment**（低延迟 REST API）、**Batch Deployment**（Pipeline 定时执行）、**Ontology 集成**（Functions 消费模型）和 **ML Ops**（CI/CD + 监控 + 反馈闭环）。

## 部署模式对比

| 模式 | 延迟 | 场景 | 交互方式 |
|------|------|------|---------|
| **Live Deployment** | 毫秒-秒级 | Workshop / Slate 实时应用 | REST API 端点 |
| **Batch Deployment** | 分钟-小时级 | Pipeline Builder 定时推理 | Python Transform |
| **Functions on Models** | 秒级 | Ontology Action 驱动推理 | TypeScript Function |
| **External Model** | 取决于外部 | 第三方平台已有模型 | API 代理 |

## Live Deployment

### 架构
```
Model Artifact
  → Modeling Objective 提交
  → 审批通过
  → Live Deployment (serverless REST API)
  → Workshop / Slate / Custom App 调用
```

### 特性
| 特性 | 说明 |
|------|------|
| Serverless | 无需管理基础设施 |
| CI/CD | 模型更新零停机 |
| 权限隔离 | 独立权限控制 |
| 资源可配 | CPU / Memory 可调 |
| 高可用 | 自动副本扩展 |

## Functions on Models（Ontology 集成）

### 两种调用方式
```typescript
// 方式 1：逐行处理 (Row-wise Processing)
@Function()
public async predictForSingleFlight(flight: Flight): Promise<Double> {
    const modelInput = {
        "lastArrivalTime": flight.lastArrivalTime,
        "lastExpectedArrivalTime": flight.lastExpectedArrivalTime,
    };
    const modelOutput = await FlightModelDeployment(modelInput);
    return modelOutput.prediction;
}

// 方式 2：批量处理 (Object Set)
@Function()
public async predictForAllFlights(flights: Flight[]): Promise<FunctionsMap<Flight, Double>> {
    const dfIn = flights.map(flight => ({
        "lastArrivalTime": flight.lastArrivalTime,
        "lastExpectedArrivalTime": flight.lastExpectedArrivalTime,
    }));
    const modelOutput = await FlightModelDeployment({"df_in": dfIn});
    // 返回 Object → prediction 映射
    return functionsMap;
}
```

### 关键参数
| 参数 | 说明 |
|------|------|
| 输入上限 | 50 MB |
| 函数超时 | 30 秒（可联系 Palantir 调整） |
| Action 编辑上限 | Function-backed Action 的编辑数量限制 |

## Batch 部署

### Python Transform 批推理
```python
from pyspark.sql import DataFrame
from transforms.api import transform

@transform(
    output=Output("ri.foundry.main.dataset.output"),
    input=Input("ri.foundry.main.dataset.input")
)
def batch_inference(input: DataFrame) -> DataFrame:
    model = load_model()
    return model.predict(input)
```

在 Pipeline Builder 中调度 → 定时执行 → 结果写回 Dataset。

## 部署决策矩阵

| 场景 | 推荐部署方式 |
|------|------------|
| Workshop 实时查询 | Live Deployment |
| 定时批量打分（如每日午夜） | Pipeline Builder + Python Transform |
| Action 触发推理（如订单下单时预测） | Functions on Models |
| 外部系统调用 | REST API (Live Deployment) |
| 第三方平台模型 | External Model Connection |

## 常见问题
**Q: Live Deployment 和 Functions on Models 何时选哪个？**
A: Workshop 用户直接查询 → Live Deployment。Ontology 对象发生变更时自动触发 → Functions on Models。

**Q: 外部模型 Batch 推理怎么做？**
A: External Model Connection 不支持 Batch。用 Python Transform 直接在 `@transform` 中调用外部 API，走 Data Connection 的 Egress Policy。
