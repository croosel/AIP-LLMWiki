# 模型集成实操

## 定义
将外部已有的 ML 模型（训练好的权重、Docker 容器、第三方平台 API）集成到 Palantir Foundry 的具体操作方法。

## 核心属性
| 集成方式 | 适用场景 |
|---------|---------|
| **模型权重导入** | 已有 `.pkl` / `.pt` / `.h5` 等训练产物 |
| **容器模型** | Docker 化的推理服务 |
| **外部 API 代理** | Azure ML / SageMaker / Vertex AI / OpenAI |
| **函数接口** | LLM 注册为 Foundry 函数（AIP Logic / Pipeline Builder） |

## 方式 1：模型权重导入 (Model Asset Files)

### 流程
```
本地/外部训练的模型权重
  → 创建 Model Artifact（File-based）
  → 上传权重文件（.pkl / .pt / .h5 等）
  → 编写 Model Adapter（定义 API 输入/输出 + predict 逻辑）
  → 提交到 Modeling Objective
```

### Model Adapter 模板
```python
import palantir_models as pm
import pickle

class MyModelAdapter(pm.ModelAdapter):
    @classmethod
    def api(cls):
        # 定义输入 Schema
        inputs = {"features": pm.Pandas(columns=[("price", float), ("exposure", int)])}
        # 定义输出 Schema
        outputs = {"prediction": pm.Pandas(columns=[("roi", float)])}
        return inputs, outputs

    def predict(self, model_input):
        # 从共享卷加载模型
        model = pickle.load(open(f"{self.shared_volume_path}/model.pkl", "rb"))
        result = model.predict(model_input["features"])
        return {"prediction": pd.DataFrame({"roi": result})}
```

## 方式 2：容器模型 (Container Model Adapter)

### 流程
```
Docker 镜像（Flask/FastAPI API 推理服务）
  → 创建 Container Model
  → 编写 ContainerModelAdapter
  → 定义 init_container 和 predict
```

### ContainerModelAdapter 模板
```python
import requests, json, pandas as pd
import palantir_models as pm

class ExampleAdapter(pm.ContainerModelAdapter):
    @classmethod
    def init_container(cls, container_context):
        shared_volume = container_context.shared_empty_dir_mount_path
        host_port = list(container_context.services.values())[0][0]
        return cls(shared_volume, host_port)

    @classmethod
    def api(cls):
        inputs = {"text": pm.String()}
        outputs = {"sentiment": pm.String()}
        return inputs, outputs

    def predict(self, model_input):
        resp = requests.post(
            f"http://{self.model_host_and_port}/predict",
            json={"text": model_input["text"]}
        )
        return {"sentiment": resp.json()["sentiment"]}
```

## 方式 3：外部 API 代理 (External Model Connection)

### 支持平台
| 平台 | 连接方式 |
|------|---------|
| OpenAI | Custom Connection + Egress Policy |
| Azure Machine Learning | 外部 API 代理 |
| Amazon SageMaker | 外部 API 代理 |
| Vertex AI | 外部 API 代理 |

### 集成步骤
```
+New → Model → Connect an externally hosted model
  → 选择 Custom Connection 源
  → 配置 Egress Policy
  → 定义 API 端点 URL + 认证凭据
  → 提交
```

### 限制
- 不支持 Batch Deployment（用 Python Transform 替代）
- 每次请求走网络，延迟高于平台内推理

## 方式 4：LLM 函数接口

### 适用场景
在 AIP Logic 的 Use LLM Block 或 Pipeline Builder 的 Use LLM Node 中调用自定义 LLM。

### 实现
```
编写 Function → 调用外部 LLM API
  → 注册为 Foundry Function
  → AIP Logic Use LLM Block 可调用
  → Pipeline Builder Use LLM Node 可调用
```
**不需要**创建 Model Artifact，直接注册为函数即可。

## 常见问题
**Q: 外部模型的 Batch 推理怎么做？**
A: 用 Python Transform 直接调用外部 API，在 Pipeline Builder 中调度执行。不通过 Model 工件。

**Q: Container Model Adapter 的 shared_volume 做什么用？**
A: 共享卷用于在训练阶段传递数据。推理阶段通常不需要，但 Adapter 可读取预加载的资源。
