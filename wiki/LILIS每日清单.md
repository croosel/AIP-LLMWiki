# LILIS 每日运营清单

## 定义
LILIS 每日运营的多 Agent 协同执行清单，按时间段划分职责，由 TL（运营总监）、Ecommerce Mind（电商思维）、国际站助手、Vibe Selling Agent 联合执行。

## 核心属性
| 属性 | 值 |
|------|-----|
| 执行团队 | TL / Ecommerce Mind / 国际站助手 / Vibe Selling Agent |
| 时间段 | 🌅 晨间 → 🚀 核心执行 → 📢 营销引流 → 🌙 晚间复盘 |
| 核心原则 | 第一性原理 / 拒绝幻觉 / 数据导向 |
| 每日定额 | 5 个产品改造 |

## 业务用途

### 🌅 晨间 (09:00 - 10:00)
- **TL**: 晨报播报，明确今日 5 个改品目标
- **Ecommerce Mind**: 检查询盘后台，锁定所有未回询盘

### 🚀 核心执行 (10:00 - 13:00)
- **Ecommerce Mind**: 5 分钟回复率保卫战，处理所有询盘，提取 Event Date
- **国际站助手**: 按方案 B 执行 5 个产品「下架→重构→重发」
  - 属性填充率必须 100%
  - 标题对标 2026 Modern Romantic 趋势
- **Ecommerce Mind**: 同步优化 SEO 详情文案

### 📢 营销引流 (14:00 - 16:00)
- **Vibe Selling Agent**: 为 Top 1 新品制作 Story 海报
- **Vibe Selling Agent**: 发布到 X 和 Instagram，带国际站链接

### 🌙 晚间复盘 (17:00 - 18:00)
- **TL**: 更新 [[LILIS运营追踪器]]
- **TL**: 检查优品晋级状态及 P4P 投放效果

## 外贸应用

### 三项执行原则
1. **第一性原理**: 所有动作服务「提升优品率」和「获取新品流量」
2. **拒绝幻觉**: 「已完成」必须有真实的 ID 变动或链接记录
3. **数据导向**: 不增加曝光或转化的动作 → 立即报告 TL 调整

## 常见问题
**Q: Event Date 是什么？**
A: 买家预定的活动日期（如婚礼日期）。提取 Event Date 可做精准复购提醒和时效推荐，是 B2B 询盘转化核心变量。

**Q: 为什么方案 B 要求属性填充率 100%？**
A: Alibaba 排序算法中将属性完整性作为优品达标硬门槛，缺 1 个必填属性即无法被评为优品。
