# Skills 技能集与 60-40 覆盖矩阵

> **目标**: 8 个 Skill 覆盖 60% 的国际站卖家高频痛点，剩下的 40% 留给 Phase 2/3 升级解决

---

## 一、8 个核心 Skill

| Skill | 触发方式 | 解决的痛点 | 依赖的 Wiki 页面 | 离钱度 |
|-------|---------|-----------|-----------------|--------|
| **p4p-diagnose** | "帮我看看 P4P 有没有浪费" | P4P 广告浪费 | P4P-Campaigns + [[P4P投流完整方法论]] | ⭐⭐⭐⭐⭐ |
| **store-health** | "店铺最近有什么问题？" | 星等级下滑 | Store-Overview + [[商家星等级]] | ⭐⭐⭐⭐⭐ |
| **product-audit** | "哪些产品需要优化？" | 低质品拉低权重 | Product-List + [[商品等级体系]] | ⭐⭐⭐⭐ |
| **listing-check** | "这个 Listing 怎么改？" | 产品描述不知怎么写 | [[标题优化黄金公式]] + [[商品卖点]] | ⭐⭐⭐ |
| **rule-qa** | "XX 规则是什么意思？" | 平台规则不理解 | 全量公共规则层 | ⭐⭐⭐ |
| **compliance-scan** | "有没有合规风险？" | 被处罚才知道缺什么 | Compliance-Status + [[实战避坑指南]] | ⭐⭐⭐⭐ |
| **quick-compare** | "我该不该提价？" | 运营决策没数据支撑 | Product-List + [[Alibaba排序逻辑]] | ⭐⭐⭐⭐ |
| **daily-push** | 定时推送日报 | 每天不知道看什么 | Store-Overview（每日更新） | ⭐⭐⭐⭐ |

### 技能设计原则

- **不追求完美自动化** —— 给出具体建议 + 依据，不自动执行（Phase 1 建立信任）
- **每个回答必须带引用** —— 指向 Wiki 中的具体页面和行，可追溯
- **回答格式统一** —— 问题 → 数据 → 分析 → 建议 → 引用

## 二、60% 痛点覆盖（当前 Phase 1）

| # | 痛点 | 覆盖 Skill | 效果预期 |
|---|------|-----------|---------|
| P1 | P4P 烧钱，不知道哪笔浪费 | p4p-diagnose | 30 天 CPA 降低 15-30% |
| P2 | 星等级下降，不知道为什么 | store-health | 24h 内定位原因 |
| P3 | 低质品不清理，影响全局 | product-audit | 当天完成全量扫描 |
| P4 | 规则太多，不知道从哪看起 | rule-qa | 即时回答，带官方引用 |
| P5 | Listing 不知道怎么优化 | listing-check | 每次回答 = 一份可执行清单 |
| P6 | AI 学习曲线陡，不知道问什么 | daily-push + store-health | 无需学习，Agent 主动推 |
| P7 | 合规风险被动发现 | compliance-scan | 提前 7-14 天发现 |

## 三、40% 痛点暂不覆盖（留给 Phase 2/3）

| 痛点 | 原因 | 何时解决 |
|------|------|---------|
| 自动执行 P4P 调价 / 改标题 | 需要信任机制和回滚能力 | [[三阶段升级路径-Sense-Reason-Act|Phase 2]] |
| 跨平台分析（Amazon/Shopify） | 当前聚焦国际站 | Phase 2+ |
| 创意内容生成（主图/视频） | 有工具但非核心 | 可优先但不主推 |
| 完整自动化运营（无人值守） | 风险和法律边界不清晰 | [[三阶段升级路径-Sense-Reason-Act|Phase 3]] |
| 竞品深度分析 | 数据源受限 | Phase 2 |

## 四、Skill 验证指标

| 指标 | Phase 1 目标 | 测量方式 |
|------|------------|---------|
| P4P 节省 | 30 天 CPA 降低 15-30% | 卖家投放数据对比 |
| 诊断准确率 | > 80% | 用户追问次数 / 采纳率 |
| 时间节省 | 运营工时减少 40% | 用户自报 + 功能使用频率 |
| 星等级改善 | 30 天至少 1 项维度改善 | 店铺星等级数据 |
| 用户留存 | 90 天留存 > 60% | 使用追踪 |
| NPS | > 40 | 产品内调查 |
