# 国际站 AI 运营 Agent 产品策略

> **产品定义**: 基于 Accio Work 平台，交付「Agent 配置包 + LLM Wiki 统一知识底座」的 B2B 运营决策产品
> **核心交付物**: 一个预配置 Agent（含 Skills + SOUL + Rules + LLM Wiki），用户导入即用
> **核心竞争力**: [[Why-Ontology]] + [[AIP-Logic]] + [[AI-FDE完整架构]] + [[本体增强生成|OAG]] → 规则 × 数据联动推理

---

## 一、产品是什么（说什么）与不是什么（不说什么）

```
┌─────────────────────────────────────────────┐
│              用户（国际站卖家）                  │
│                      ↓                       │
│            Accio Work 平台（宿主）               │
│                      ↓                       │
│        ┌──────────────────────────┐          │
│        │  预配置 Agent（交付物）      │          │
│        │  ├── Skills（运营技能集）    │          │
│        │  ├── SOUL（运营顾问人格）    │          │
│        │  └── Rules（行为约束）       │          │
│        ├──────────────────────────┤          │
│        │  LLM Wiki（统一知识底座）     │          │
│        │  ├── 公共规则层（国际站知识）  │          │
│        │  ├── 店铺数据层（私有数据）    │          │
│        │  └── [[双向链接]]推理层       │          │
│        └──────────────────────────┘          │
│                      ↓                       │
│              MCP 连接器（阿里国际站 API）         │
└─────────────────────────────────────────────┘
```

### 产品不是什么（修正）

| 误解 | 修正 |
|------|------|
| 建一个独立 Chatbot Web App | 利用 Accio Work 现有 Agent 框架 |
| 开发一套 SaaS 系统 | 交付 Agent 配置包 + LLM Wiki |
| 自己写 UI/支付/认证 | 宿主平台提供基础设施 |
| 做一个"软件产品" | 做一个"Agent 模板产品" |

### 三个核心设计决策

| 决策 | 说明 |
|------|------|
| **LLM Wiki 是唯一底座** | 所有知识（规则 + 店铺数据）都存入 LLM Wiki，Agent 直接读取。不需要 RAG 管道、向量数据库、独立知识库 |
| **Agent + Skills 是交付物** | 用户买的不是软件，是一个"配好技能的 AI 运营顾问" |
| **[[60-40痛点覆盖矩阵|60% 目标]]** | 只做"离钱最近"的高频痛点：P4P 诊断、店铺健康检查、低质品告警、规则问答 |

## 二、产品架构（AIP 四层模型）

| 层 | 对应 AIP 组件 | 功能 | 技术实现 |
|----|-------------|------|---------|
| **Sense 感知层** | [[AIP-Observability]] + MCP | 阿里国际站数据实时同步 → LLM Wiki | MCP Connector + Cron 定时任务 |
| **Reason 推理层** | [[AIP-Logic]] + [[Ontology]] + [[AI-FDE完整架构|AI-FDE]] | 规则引擎 + 知识图谱 + LLM 推理 | AIP Logic 自动化管道 + OAG |
| **Act 执行层** | Automate + [[动力层与写回]] | 手动 → 半托管 → 全托管 | Skills 触发 + Logic Staged/Auto-apply |
| **治理层** | [[AIP-Evals]] + [[AIP-伦理与治理]] | 质量评估 + 幻觉检测 + 操作审计 | Evals 套件 + Audit-Log |

详见 [[三阶段升级路径-Sense-Reason-Act]]。

## 三、为什么不是 SaaS 而是 Agent 模板

| 维度 | SaaS 路径 | Agent 模板路径 |
|------|----------|---------------|
| 开发成本 | $50K+（UI/后端/支付/认证） | ~$0（Accio Work 承担） |
| 维护成本 | 服务器 + 运维 $850-2,850/月 | $120-400/月 |
| 分发方式 | 自己获客 | 手动配置交付（未来：平台市场） |
| 核心壁垒 | 软件功能 | [[经验累积与竞争壁垒|知识深度 + 数据联动 + 经验累积]] |
| 收⼊模型 | 订阅费（低 ARPU） | 服务费（配置 + 知识运维） |
| 升级路径 | 需要大量重新开发 | [[三阶段升级路径-Sense-Reason-Act|Sense → Reason → Act]] 自然演化 |
