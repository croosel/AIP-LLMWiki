# Alibaba 国际站排序逻辑

## 定义

Alibaba 国际站的搜索排序引擎由 **显性权重**（销量驱动）和 **隐性权重**（相关性匹配）双系统构成，辅以**量率模型**（转化量+转化率）双向考核。理解排序逻辑是店铺运营的底层基础。

## 核心属性

### 1. 显性权重 — 销量驱动模型

| 规则 | 细节 | 操作启示 |
|------|------|---------|
| 销量权重独立 | 与商品价格无关，仅看成交笔数 | 打造爆品 > 追求高客单价 |
| 信保底价 | 信保订单实际成交价 ≥ 页面价的 50% | 不能低于此线，否则成交不计销量 |
| 低价引流 | 新店&新品初期可用低价引流 | 最低价 = 可实际成交的最低价，杜绝成交不卖 |
| 180天最低价 | 页面价、折扣价、成交价三者取最低 | 需小心折扣策略对长期权重的影响 |
| 买家数 | 90天内同一买家多笔订单计为 1 | 复购要做但核心是拓展买家数 |
| 数据周期 | 近 60/90/180 天数据 | 180 天为最大权重因子 |

### 2. 隐性权重 — 相关性匹配模型

| 维度 | 传统模式 | AI Mode（2024+） |
|------|---------|-----------------|
| 匹配方式 | 关键词匹配 | 全局语义匹配 |
| [[标题优化黄金公式|标题]] | 128 字符，精准堆词 | 128 字符 + **动态标题** |
| [[商品卖点]] | 无 | **2000 字符**，进入搜索索引 |
| 属性/SKU | 不参与匹配 | **全量参与搜索匹配**（包含规格属性图） |
| 重铺诊断 | 关键词查重 | **综合判重**（主/副图+标题+属性） |
| 商品正文 | 有限参与 | **全部参与搜索匹配**（推荐结构化商详） |
| 千人千面 | 无 | 根据买家偏密调用**规格属性图片**替换主图 |

> **核心变化**: 标题不再是唯一匹配入口，商品一切文本信息都可被索引。商品卖点的 2000 字符空间是最被低估的 SEO 金矿。

### 3. 量率模型 — 双轮驱动

```
        量率模型
       /       \
 转化量          转化率
 /    |    \      |
曝光量 点击量 商机量  点击率
成交量 评价量 复购量  商机转化率
                 成交率
                 好评率
                 复购率
```

| 周期 | 作用 |
|------|------|
| 近 60 天 | 短期趋势，判断运营动作效果 |
| 近 90 天 | 中期沉淀，[[商家星等级|星等级]] 评定核心周期 |
| 近 180 天 | 长期积累，搜索排序最大权重因子 |

## 业务用途

- 制定定价策略（新店低价引流 vs 成熟期价值定价）
- 评估标题和卖点的 SEO 优化方向
- 结算周期的权重分配影响运营节奏

## 常见问题

**Q: 为什么我的销量增加了但排名没变化？**
A: 销量权重基于近 180 天数据，新增加的销量需要时间沉淀。同时需检查信保成交价是否 ≥ 页面价 50%。

**Q: AI Mode 下还需要做关键词吗？**
A: 需要，但策略变了——不再堆砌关键词，而是全面优化所有文本字段（标题 128 + 卖点 2000 + 属性 + SKU + 正文）。

**Q: 折扣价会影响长期权重吗？**
A: 会。180 天最低价取页面价/折扣价/成交价三者最低，频繁折扣可能拉低长期权重基线。
